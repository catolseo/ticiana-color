# Расшифровка базы формул AdsPro

Инструкция для синхронизации формул `ticiana-color` с боевой машиной AdsPro.

## TL;DR

```bash
# 1. Расшифровать боевые CSV в data/book/
python tools/decrypt_book.py "D:/backup/Ticiana/AdsPro/book" data/book

# 2. Скопировать актуальный каталог продуктов
cp data/book/tint_book.ini data/tint_book.ini

# 3. Перегенерировать data.js + formulas/p*.js
python tools/extract_csv.py

# 4. Закоммитить и запушить
git add -A && git commit -m "Sync formulas from AdsPro $(date +%y%m%d)" && git push
```

GitHub Pages подхватит изменения автоматически. Cache-bust в `index.html`
завязан на content-hash, так что пользователи увидят новые данные сразу.

## Алгоритм шифра

```
файл = <4 байта magic prefix> + <зашифрованный поток>
key[i] = (7 + 7 * i) mod 255
plain[i] = cipher[i + 4] XOR key[i]
```

Код — [`tools/decrypt_book.py`](decrypt_book.py), 30 строк.

### Особенности

- **Magic prefix** `c0 c4 bf 3a` — одинаковый у всех 200+ файлов, версионный
  маркер формата. Отбрасывается перед XOR.
- **mod 255**, не mod 256. Wraparound каждые 36-37 байт; при mod-256 это
  выглядит как «+1 глитч в +7 прогрессии» и на нём ломается константный XOR.
- **k0 = 7** одинаковый для всех файлов — per-file derivation нет. Имя
  файла, его длина, дата сохранения — никакой роли в ключе не играют.
- Шифр потоковый, размер cipher = размер plaintext + 4 байта magic.

### Как был взломан

Known-plaintext атака: у нас были одни и те же файлы и в открытом виде
(Jan-2024 backup), и в зашифрованном (Dec-2023 production снапшот).
XOR cipher с plain дал предполагаемый keystream. В нём виден +7 шаг,
прерывающийся периодически на «+1 глитчах». Глитчи оказались точками
wraparound по mod-255 — единственному модулю, при котором +7-прогрессия
складывается в идеальную последовательность без разрывов.

Полный лог исследования и pyhton-эксперименты см. в git-истории коммита
`7725e26` (там же удалённые `decrypt_attempt*.py` debug-скрипты).

## Что входит в обновление

После расшифровки получаем 206 файлов формул. `extract_csv.py` парсит
их вместе с `tint_book.ini` и выдаёт:

- `data.js` — каталог продуктов, баз, колорантов (~38 КБ)
- `formulas/p<N>.js` — формулы одной продуктовой линии, ~91 файл
- Цифры: 91 продукт, ~109 000 формул, 16 колорантов

Версия в шапке UI собирается автоматически из `LoadDate` в ini —
например `AdsPro 03.07.2025 · 91 продуктов · 108 985 формул`.

## Когда повторно синхронизироваться

- После любого изменения формул в AdsPro (новые продукты, исправления,
  свежие колеровки).
- Когда в `tint_book.ini` вырастает `LoadDate`.
- Если технолог говорит «вчера добавили линейку X» — берём свежий бэкап
  боевой папки и запускаем шаги из TL;DR.

Если шифр вдруг изменится (Hongpu выпустит новую версию AdsPro), magic
prefix `c0 c4 bf 3a` или сломанный `(7 + 7·i) mod 255` keystream это
покажут — `decrypt_book.py` выдаст мусор. В таком случае повторить
known-plaintext атаку из имеющегося snapshot'а.
